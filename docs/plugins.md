# 🔌 نظام الإضافات المقترح (Plugins Architecture Specification)

يوضح هذا المستند المواصفة التقنية وتصميم **نظام الإضافات (Plugins System)** المقترح لنظام **Beidar**، والذي يهدف لتمكين المطورين من توسيع قدرات النظام ديناميكياً على بيئة ويندوز دون تعديل النواة.

---

## 1. محرك الباكيند (Backend Extensions Engine)

بسبب القيود التقنية لبيئة ويندوز مع حزمة Go `plugin` الرسمية، يعتمد تصميم الباكيند على طريقتين اختياريتين حسب طبيعة وحجم الإضافة:

### أ. محرك الأكواد النصية المضمنة (Embedded Scripting via Goja)
للإضافات الخفيفة والمتوسطة، يتم تضمين محرك **`goja`** (مترجم JavaScript مكتوب بالكامل بـ Go) داخل التطبيق لتشغيل منطق الإضافة المكتوب بـ JS في بيئة معزولة:
* **تسجيل الواجهات (APIs Export)**: يقدم التطبيق الرئيسي كائن `BeidarAPI` للمحرك يحتوي على دوال مراقبة ومحمية مثل (`GetInventory()`, `CreatePayment()`, `PrintReceipt()`).
* **دورة الحياة (Lifecycle)**: يقوم محرك Go عند التشغيل بمسح مجلد `plugins/` وقراءة ملفات `backend.js` المرفقة وتمريرها للمفسر لتسجيل المستمعين (Event Listeners).

### ب. العمليات المنفصلة (gRPC Out-of-Process Plugins)
للتكاملات المعقدة أو العمليات الكثيفة حسابياً، تعمل الإضافة كبرنامج تنفيذي مستقل (`.exe`) يوضع في مجلد الإضافة:
* **الاتصال**: عند بدء التشغيل، يقوم Beidar بإنشاء عملية فرعية (Subprocess) للإضافة ويمرر لها منفذ عشوائي (Random Port).
* **التخاطب**: يتم تبادل البيانات واستدعاء الدوال عن بعد (RPC) باستخدام **gRPC** عبر بروتوكول HTTP/2 محلي.
* **الحماية**: إذا تعرضت الإضافة المنفصلة لعطل أو انهيار، تقوم النواة بإعادة تشغيلها تلقائياً دون تأثر برنامج POS الرئيسي.

---

## 2. سجل الإضافات والتحميل الديناميكي في React (Frontend Registry)

تعتمد الواجهة الأمامية على تحميل واجهات المستخدم الرسومية للإضافات ديناميكياً في وقت التشغيل (Runtime Dynamic Loading):

1. **التحميل الديناميكي (Dynamic Script Injection)**:
   عند تشغيل الواجهة، يقوم نظام الملحقات بقراءة ملفات `plugin.json` لمعرفة الإضافات النشطة، ويقوم بحقن ملف `ui.js` الخاص بكل إضافة ديناميكياً داخل كائن الـ DOM:
   ```javascript
   const script = document.createElement('script');
   script.src = `wails://plugins/${pluginId}/ui.js`;
   document.head.appendChild(script);
   ```

2. **سجل تسجيل الإضافات (Global Registry)**:
   يعرّف التطبيق الرئيسي كائناً عالمياً في المتصفح `window.BeidarPlugins` يحتوي على دالة التسجيل `registerPlugin(pluginConfig)`:
   ```typescript
   interface PluginConfig {
       id: string;
       name: string;
       views: {
           route: string;
           component: React.ComponentType;
           title: string;
           icon: string;
       }[];
       hooks: {
           target: 'pos-sidebar' | 'payment-methods' | 'settings-panel';
           component: React.ComponentType;
       }[];
   }
   ```

## 3. نقاط الحقن ومواضع الإلحاق (Interface Hooks)

لتضمين عناصر الإضافة الرسومية في واجهة Beidar، يتم توفير **نقاط حقن (Hooks)** استراتيجية في شاشات التطبيق:

* **حقن القائمة الجانبية (`pos-sidebar`)**:
  تسمح للإضافة بإضافة خيار قائمة جديد (Tab) يؤدي إلى شاشة كاملة خاصة بالإضافة (مثل: شاشة إدارة وحجز الطاولات للمطاعم).
* **حقن طرق الدفع (`payment-methods`)**:
  تتيح للإضافة إلحاق خيار دفع جديد في شاشة إتمام البيع (مثل: الدفع عبر محفظة إلكترونية خارجية أو بوابة دفع بالبطاقات الائتمانية).
* **حقن لوحة الإعدادات (`settings-panel`)**:
  تسمح للإضافة بإلحاق خيارات تكوين خاصة بها داخل شاشة إعدادات التطبيق لضبط مفاتيح الربط أو الإعدادات المحلية.

## 4. الأمان وتحديد صلاحيات الإضافة (Security & Sandboxing)

لحماية بيانات نقاط البيع الحساسة من السرقة أو التخريب، تتبع الإضافات نموذج حماية صارم:

* **ملف التعريف والمطالبة بالصلاحيات (`plugin.json`)**:
  يجب على كل إضافة الإفصاح عن الصلاحيات التي تحتاجها بشكل مسبق، ولا يتم منحها أي صلاحية خارج هذا التصريح:
  ```json
  {
      "id": "delivery-sync",
      "permissions": [
          "read:orders",
          "write:orders",
          "network:outbound"
      ]
  }
  ```
* **عزل الشبكة (Network Restriction)**:
  بشكل افتراضي، يُمنع مفسر جافا سكريبت `goja` من إجراء أي اتصالات شبكة خارجية إلا إذا تم التصريح بـ `network:outbound` مع تحديد النطاقات الموثوقة (Domains Whitelist).
* **عزل الملفات (File System Jail)**:
  لا تملك الإضافة حق قراءة أو كتابة أي ملفات خارج مجلد البيانات الخاص بها (`/appdata/plugins/data/plugin_id/`).
